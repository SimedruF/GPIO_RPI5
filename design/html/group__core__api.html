int gpio_init ( void  )

Initialize the GPIO subsystem.

Must be called once before any other GPIO function. Opens the memory-mapped device (mmap backend) or prepares internal state (pinctrl backend).

Returns
0 on success, -1 on failure (e.g., /dev/gpiomem0 not accessible).

Initialize the GPIO subsystem.

Opens /dev/gpiomem0 and maps 192 KB of RP1 register space into the process address space. All pin states are reset to UNDEF.

Must be called exactly once before any other GPIO function.

Returns
0 on success, -1 if the device cannot be opened or mmap fails.
Note
Requires read/write access to /dev/gpiomem0 (default on Raspberry Pi OS).

Initialize the GPIO subsystem.

Resets all pin states to UNDEF and sets the initialization flag. Unlike the mmap backend, no device files are opened — each subsequent operation spawns a pinctrl CLI process.

Returns
Always returns 0 (cannot fail).

pin_t pinopen ( int  pin,
int  mode 
)

Open a GPIO pin for use in the specified direction.

Configures the pin as INPUT or OUTPUT by setting the FUNCSEL to SIO mode and adjusting the output-enable register. The pin must be opened before calling pinwrite(), pinread(), or pintoggle().

Parameters
pinGPIO number (0–53 for mmap backend, 0–237 for pinctrl).
modeDirection: INPUT (0) or OUTPUT (1).
Returns
A pin_t struct with the configured mode. On error, both fields are UNDEF.

Open a GPIO pin for use in the specified direction.

Configures the pin for standard GPIO (SIO) operation by:

  1. Setting FUNCSEL to SIO (ALT5) in the IO_BANK CTRL register.
  2. Clearing output/OE overrides so SIO controls the pin.
  3. Enabling input (IE) and clearing output-disable (OD) in the pad register.
  4. Setting or clearing the Output Enable bit via atomic RIO registers.
Parameters
pinGPIO number (0–53).
modeDirection: INPUT (0) or OUTPUT (1).
Returns
pin_t with the configured mode. On error, both fields are UNDEF.

Open a GPIO pin for use in the specified direction.

Runs pinctrl set <pin> op (output) or pinctrl set <pin> ip (input) via popen(). The command spawns a separate process, making this ~1000x slower than the mmap backend.

Parameters
pinGPIO number (0–53, 100–135, or 200–237).
modeDirection: INPUT (0) or OUTPUT (1).
Returns
pin_t with the configured mode. On error, both fields are UNDEF.

Close a GPIO pin and reset it to a safe state.

Sets the pin back to input mode and disconnects the function selector (FUNCSEL = NULL). The internal state is reset to UNDEF.

Close a GPIO pin and reset it to a safe state.

Clears the Output Enable bit (sets pin to input) and sets FUNCSEL to GPIO_FUNC_NULL (31) to disconnect the pin from all peripherals. The cached state is reset to UNDEF.

Parameters
indx_pinGPIO number (0–53) to close.

Close a GPIO pin and reset it to a safe state.

Runs pinctrl set <pin> ip to reset the pin to a safe input state. The cached mode and state are reset to UNDEF.

Parameters
indx_pinGPIO number to close.

Read the current logic level of a GPIO pin.

Reads the RIO_IN input register and extracts the bit for the given pin. Works on both INPUT and OUTPUT pins (output readback is supported).

Parameters
pinGPIO number (0–53).
Returns
HIGH (1), LOW (0), or -1 on error.

Read the current logic level of a GPIO pin.

Runs pinctrl get <pin> and parses the output for "| lo" or "| hi" to determine the logic level. The pipe delimiter avoids false matches with pin name substrings.

Parameters
pinGPIO number.
Returns
HIGH (1), LOW (0), or -1 on error or if the level cannot be parsed.

Toggle the output state of a GPIO pin.

Uses the atomic XOR register on SYS_RIO to flip the output bit in a single write. This is faster and race-free compared to read-modify-write.

0 on success, -1 on error.

Configure the internal pull resistor for a GPIO pin.

Sets pull-up, pull-down, or no-pull on the pin's pad register. Pull resistors ensure a defined logic level when the pin is not actively driven — essential for buttons, open-drain buses, etc.

Parameters

0 on success, -1 on error.

Configure the internal pull resistor for a GPIO pin.

Modifies the PUE (pull-up enable) and PDE (pull-down enable) bits in the pad register. Both bits are cleared first, then the requested pull is set.

Parameters
pinGPIO number (0–53).
pullPull mode: PULL_UP (1), PULL_DOWN (2), or PULL_NONE (0).

0 on success, -1 on error (invalid pin, not initialized, or bad pull value).

Set the output drive strength for a GPIO pin.

Configures how much current the pin can source or sink. Higher values are needed for driving long traces, LEDs, or fast digital signals.

Parameters
pinGPIO number.
strengthOne of DRIVE_2MA, DRIVE_4MA, DRIVE_8MA, or DRIVE_12MA.
Returns
0 on success, -1 on error.
Note
Only supported by the mmap backend. The pinctrl backend returns -1.

Set the output drive strength for a GPIO pin.

Modifies the DRIVE field (bits 5:4) in the pad register to control the maximum source/sink current. Higher drive is useful for fast signals (SPI clock) or pins driving long traces.

Parameters
pinGPIO number (0–53).
strengthOne of: DRIVE_2MA (0), DRIVE_4MA (1), DRIVE_8MA (2), DRIVE_12MA (3).
Returns
0 on success, -1 on error.

Set the output drive strength for a GPIO pin.

The pinctrl CLI does not expose drive strength control. Use the mmap backend (gpio_rpi5.c) if drive strength configuration is required.

Parameters
pinIgnored.
strengthIgnored.
Returns
Always returns -1.
